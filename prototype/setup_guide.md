# GPS World Lap Time Counter 導入手順

構成: HTML1枚（参加者のスマホで開く）＋ GAS ＋ スプレッドシート

```
参加者スマホ(iPhone/Android ブラウザ)
   └─ 内蔵GPSで計測ライン通過を検知
        └─ ラップ確定ごとにGASへ自動送信
             └─ スプレッドシートに蓄積
                  ├─ ランキングシート(自動集計)
                  └─ 印刷用シート(A4でそのまま出力)
```

---

## 手順1: スプレッドシート側（10分）

1. Googleスプレッドシートを新規作成（名前例:「GPSタイム計測 2026」）
2. メニュー「拡張機能」→「Apps Script」
3. `gas_backend.gs` の中身を全部貼り付けて保存
4. 関数選択で **setup** を選び「実行」（初回は権限承認あり）
   → RawLaps / ランキング / 印刷用 の3シートが自動作成される
5. 「デプロイ」→「新しいデプロイ」→ 種類:**ウェブアプリ**
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**
6. 発行された `https://script.google.com/macros/s/…/exec` をコピー

## 手順2: 計測アプリの設置（5分）

`lap_timer.html` をHTTPSで配信できる場所に置く。無料なら:

- **GitHub Pages**（おすすめ・RATIO LIFINGと別管理できる）
- Netlify Drop（ドラッグ&ドロップだけ）

※ ブラウザのGPS機能はHTTPS必須のため、ファイルを直接開く形（file://）では動きません。

置いたURLをQRコードにして受付で配布すると楽です。

## 手順3: 参加者側の操作

1. QR/URLからページを開く
2. 「設定」タブ → 名前・車両・送信先URL（手順1のURL）を入力
   ※送信先URLは主催者側で案内文にコピペ配布
3. コントロールライン設定（下記）
4. 「設定を保存」
5. 「計測」タブ → スマホをダッシュボードに固定 → 計測スタート

## コントロールラインの設定方法

計測の基準となる「コースを横切る仮想の線」を2点で決めます。

- 主催者が事前に、ホームストレートの**コース左端の外側**でA点、**右端の外側**でB点を「現在地取得」ボタンで取得
- 取得した数値（lat,lng）を参加者全員に配布 → 各自が設定画面に貼り付け
- **全員が同じ数値を使うこと**（人によってラインが違うと不公平になるため)

## 運用時の注意

- **画面は点灯したまま**にする（消えるとGPSが止まる。アプリ側で点灯維持をかけるが、iPhoneの自動ロックは「なし」推奨）
- スマホはフロントガラス下・ダッシュボード上に固定（GPS感度が最も良い）
- 機内モードにしない（送信に電波が必要。圏外時は復帰後に自動再送）
- 精度は±0.2〜0.5秒程度。お楽しみ企画向け。賞典で0.1秒を競う場合はフェーズ2（外部GPS対応）を検討

## 印刷

スプレッドシートの「印刷用」シートを開き、ファイル→印刷→A4縦でそのまま出力できます。日付は自動で当日になります。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| GPSエラーが出る | ブラウザの位置情報許可を確認（設定→Safari/Chrome→位置情報） |
| ラップが記録されない | コントロールラインの2点がコース幅を完全にまたいでいるか確認 |
| 1周ごとに2回カウントされる | 最小ラップタイムをコースの最速想定タイムの8割程度に上げる |
| ランキングが取得できない | GASのデプロイ設定が「全員」になっているか確認 |
| シートに届かない | 送信先URLが `/exec` で終わっているか確認（/devは不可） |
